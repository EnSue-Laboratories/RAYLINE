# Dispatch Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a `Dispatch` sidebar button that opens a modal letting the user spawn N chats in parallel — each in its own worktree/branch — either from a GitHub issue list or a user-authored custom list. Per-row prompt, branch name, model override, and (custom mode) attachments. One global model picker applies as default.

**Architecture:** Two new IPC exposures on `window.api` (`ghListIssues`, `gitRemoteSlug`) reuse the existing `gh-list-issues` handler and add a small `git remote` wrapper. A new `<DispatchCard>` React component renders the modal. A new `handleDispatch` orchestrator in `App.jsx` calls the existing `handleCreateChat` N times in parallel via `Promise.allSettled`. Conversation drafts gain optional `dispatchId` + `tags` fields; the sidebar renders tags as small pills.

**Tech Stack:** React 19, Electron 41 (preload IPC bridge), lucide-react icons, `git` CLI for remote parsing, `gh` CLI via `github-manager.cjs`.

**Reference:** `docs/plans/2026-04-19-dispatch-design.md`

**Verification policy:** This project has no unit tests for UI/IPC. Each task ends with `npm run lint` and, when applicable, a manual smoke step in the running dev app (`npm run dev:electron`).

**Commit style:** Follow existing conventional-commit pattern (`feat:`, `chore:`, `fix:`). One commit per task.

ipcMain.handle("git-remote-slug", async (_event, cwd) => {
  if (!cwd) return null;
  try {
    const raw = await git(["remote", "get-url", "origin"], cwd);
    // Match "git@github.com:owner/repo.git" or "https://github.com/owner/repo(.git)?"
    const m = raw.trim().match(/github\.com[:/]([^/]+)\/([^/.\s]+?)(?:\.git)?$/);

### Task 3: Add `handleDispatch` orchestrator in App.jsx

**Why:** The modal needs a single entry point that validates rows, generates a shared `dispatchId`, fires N `handleCreateChat` calls in parallel, and returns a per-row result summary so the UI can show partial failures.

**Files:**
- Modify: `src/App.jsx` (add new function directly below `handleCreateChat`, around `App.jsx:2057`)

**Step 1: Add the orchestrator**

Insert after the `handleCreateChat` declaration closing at `App.jsx:2056`:

```javascript
const handleDispatch = useCallback(async (rows) => {
  // rows: Array<{ prompt, attachments?, model?, cwd, branch, issueContext?, tag? }>
  const dispatchId = "d" + Date.now() + "-" + Math.random().toString(36).slice(2, 8);
  const tasks = rows.map((row) => ({
    row,
    promise: handleCreateChat({
      prompt: row.prompt,
      attachments: row.attachments,
      model: row.model || defaultModel,
      cwd: row.cwd,
      worktree: true,
      branch: row.branch,
      issueContext: row.issueContext,
      dispatchId,
      tags: ["dispatch", ...(row.tag ? [row.tag] : [])],
    }).then(() => ({ ok: true, row })).catch((err) => ({ ok: false, row, error: err })),
  }));

  const results = await Promise.all(tasks.map((t) => t.promise));
  return { dispatchId, results };
}, [handleCreateChat, defaultModel]);
```

Note: `handleCreateChat` currently does not return the chat id. This is fine — the orchestrator only needs success/failure to report partial outcomes. If the id becomes useful later, `handleCreateChat` can be extended to return `id` at the very end.

function slugifyBranch(text, fallbackIndex) {
  const slug = (text || "")
    .toLowerCase()
    .replace(/[^a-z0-9]+/g, "-")
    .replace(/(^-+|-+$)/g, "")
    .slice(0, 32);
  return slug || `task-${fallbackIndex + 1}`;
}

function defaultCustomBranch(index) {
  const d = new Date();
  const yyyy = d.getFullYear();
  const mm = String(d.getMonth() + 1).padStart(2, "0");
  const dd = String(d.getDate()).padStart(2, "0");
  const hh = String(d.getHours()).padStart(2, "0");
  const mi = String(d.getMinutes()).padStart(2, "0");
  return `dispatch-${yyyy}${mm}${dd}-${hh}${mi}-${index + 1}`;
}

function makeCustomRow(index) {
  return {
    key: "r" + Date.now() + "-" + index,
    prompt: "",
    branch: defaultCustomBranch(index),
    model: "",           // "" = use global
    attachments: [],
  };
}

function IssueTab({ rows, setRows, projects, currentCwd, availableModels, errors }) {
  const [selectedPath, setSelectedPath] = React.useState(currentCwd || "");
  const [slug, setSlug] = React.useState(null);
  const [loading, setLoading] = React.useState(false);
  const [loadError, setLoadError] = React.useState(null);

  const projectPaths = React.useMemo(
    () => Object.keys(projects || {}).filter((p) => !projects[p]?.hidden),
    [projects]
  );

  React.useEffect(() => {
    if (!selectedPath) { setSlug(null); setRows([]); return; }
    let cancelled = false;
    setLoading(true); setLoadError(null);
    (async () => {
      try {
        const s = await window.api.gitRemoteSlug(selectedPath);
        if (cancelled) return;
        if (!s) {
          setSlug(null); setRows([]); setLoadError("No GitHub remote on this folder.");
          setLoading(false); return;
        }
        setSlug(s);
        const issues = await window.api.ghListIssues(s, "open");
        if (cancelled) return;
        setRows((issues || []).map(makeIssueRow));

        if (!cancelled) setLoadError(e?.message || "Failed to load issues.");
      } finally {
        if (!cancelled) setLoading(false);
      }
    })();
    return () => { cancelled = true; };
  }, [selectedPath, setRows]);

  const updateRow = (key, patch) => setRows((prev) =>
    prev.map((r) => (r.key === key ? { ...r, ...patch } : r))
  );

  return (
    <div style={{ padding: 14 }}>
      <div style={{ display: "flex", alignItems: "center", gap: 10, marginBottom: 10 }}>
        <label style={{ fontSize: 12, color: "rgba(255,255,255,0.6)" }}>Repo folder</label>

          value={selectedPath}
          onChange={(e) => setSelectedPath(e.target.value)}

const handleSubmit = useCallback(async () => {
  const rowsToRun = tab === "issues"
    ? issueRows.filter((r) => r.enabled)
    : customRows;

  // Build per-row errors map
  const rowErrors = {};
  const seenBranches = new Set();
  for (const r of rowsToRun) {

    else if (!r.branch?.trim()) rowErrors[r.key] = "Branch name is empty.";
    else if (seenBranches.has(r.branch)) rowErrors[r.key] = "Branch name is duplicated in this batch.";

        null : null)

    rowErrors.__global = "Select a folder in the sidebar first.";
  }

  if (Object.keys(rowErrors).length) {
    setErrors(rowErrors);
    return;
  }
  setErrors({});
  setSubmitting(true);

  const payload = rowsToRun.map((r) => ({
    prompt: r.prompt,
    attachments: r.attachments,
    model: r.model || globalModel,
    cwd: r.cwd || currentCwd,
    branch: r.branch.trim(),
    issueContext: r.issue ? `Issue #${r.issue.number}: ${r.issue.title}` : undefined,
    tag: r.issue ? `#${r.issue.number}` : undefined,
  }));

  const { results } = await onDispatch(payload);
  const failed = results.filter((x) => !x.ok);
  if (failed.length === 0) {
    onClose();
    return;
  }

  // Keep failed rows in modal; mark errors per row.
  const failedMsgs = {};
  for (const f of failed) {
    failedMsgs[f.row.__rowKey || f.row.branch] = f.error?.message || "Dispatch failed.";
  }
  // Note: results' `row` is the payload row, which lacks `key`; reconstruct by branch.
  const byBranch = {};
  rowsToRun.forEach((r) => { byBranch[r.branch.trim()] = r.key; });
  const keyedErrors = {};
  for (const f of failed) {
    const key = byBranch[f.row.branch];
    if (key) keyedErrors[key] = f.error?.message || "Dispatch failed.";
  }
  setErrors(keyedErrors);

  if (tab === "issues") {
    // Turn off the `enabled` flag on successful rows so only failures remain armed.
    const successBranches = new Set(results.filter((x) => x.ok).map((x) => x.row.branch));
    setIssueRows((prev) => prev.map((r) => successBranches.has(r.branch) ? { ...r, enabled: false } : r));
  } else {
    // Remove successful custom rows entirely.
    const successBranches = new Set(results.filter((x) => x.ok).map((x) => x.row.branch));
    setCustomRows((prev) => prev.filter((r) => !successBranches.has(r.branch.trim())));
  }

  setSubmitting(false);
}, [tab, issueRows, customRows, currentCwd, globalModel, onDispatch, onClose]);
